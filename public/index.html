How this works, honestly. Layer 1 is a deterministic keyword engine over a small illustrative rule set — instant, offline, fully reproducible. Layer 2 sends the draft to an LLM to catch what keywords miss (tone, implied advice, missing context); it reasons over the rule summaries it's given, it does not know today's regulations from memory, and it is not a source of legal truth. Layer 3 is the only genuinely live regulatory data in this prototype — a real-time query to the FCA's public Financial Services Register to check UK authorisation status. There is no free API that returns marketing-rule text itself; that knowledge base is maintained by Legal & Compliance and would need to stay human-owned even in production. Nothing here is legal advice, and every red or amber flag should route to a human reviewer.
